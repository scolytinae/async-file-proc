# Задача — Асинхронный файловый обработчик

Напиши систему которая обрабатывает папку с текстовыми файлами в несколько этапов.

Что нужно сделать:

Сначала запусти `generator.py` — он создаёт 20 текстовых файлов со случайным текстом. Дальше пишешь основную систему.

## Этап 1 — Чтение файлов

Читаем файлы через ThreadPoolExecutor + run_in_executor. Одновременно не более 3 файлов — Semaphore. Прочитанные файлы кладём в Queue.

## Этап 2 — Обработка

Consumer читает из Queue и обрабатывает каждый файл через ProcessPoolExecutor + run_in_executor. Обработка — CPU-bound задача: считаем количество слов, топ-10 самых частых слов, среднюю длину слова. Если обработка висит больше 5 секунд — пропускаем файл (`wait_for` + `TimeoutError`).

## Этап 3 — Сборка результатов

Lock защищает общий счётчик обработанных файлов. Event сигналит что все файлы прочитаны и consumer может заканчивать. В конце сохраняем результат в JSON.

## Требования:

- TaskGroup запускает readers и processors параллельно
- reader и processor работают конвейером через Queue — пока один файл обрабатывается, следующий уже читается
- Pydantic-модель для результата обработки каждого файла
- В итоговом JSON — статистика по каждому файлу + общая сводка (всего слов, самые частые слова по всем файлам)

Структура:

```
file_processor/
├── generator.py    # генерируем 20 тестовых файлов
├── reader.py       # ThreadPoolExecutor — читаем файлы
├── processor.py    # ProcessPoolExecutor — считаем статистику
├── storage.py      # пишем результат в JSON
└── main.py         # TaskGroup, Queue, Semaphore, Event, Lock
```
